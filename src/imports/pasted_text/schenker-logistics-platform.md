SCHENKER

Simplifying a complex shipping workflow

Designing a structured logistics interface for shipment creation, document handling, and operational efficiency.

Overview

SCHENKER is a desktop shipping management platform built for operational logistics tasks.

The system brings together shipment creation, sender and receiver data, package setup, labels and tracking, packing lists, waybills, settings, and technical tools inside one workflow.

The goal was to make this complexity feel clear, structured, and manageable in daily use.

The Challenge

Enterprise logistics systems often become difficult to use because they combine many repetitive, detail-heavy tasks in one place.

This project needed to support:

multi-step shipment setup

large amounts of form input

document-heavy workflows

operational speed

technical setup and configuration

The challenge was to reduce friction without reducing capability.

What makes this work stand out

What sets this design apart is the ability to turn operational complexity into a clear working system.

Instead of focusing only on visual polish, the project demonstrates:

strong workflow thinking

clear information architecture

structured form design

consistency across complex modules

usability for repeated daily tasks

This is a product built for real use, not just presentation.

My Role

My focus in this project was to design a usable structure for a dense enterprise environment.

I worked on:

workflow organization

modular product structure

form-heavy screen design

operational table layouts

consistency across key product areas

The main objective was to help users complete tasks faster and with more confidence.

Product Structure

The platform is organized into focused modules:

Shipment

Labels and Tracking

Packing List

Waybill

Settings

Help

Developer Tools

This modular structure improves clarity and helps users move through the product without confusion.

Shipment Flow

The shipment creation process is the core workflow of the product.

Instead of placing everything on one screen, the flow is divided into clear steps:

shipment data

information

sender

receiver

packages

This step-based approach reduces cognitive load and makes complex data entry easier to complete.

Designing for Form-Heavy Tasks

A large part of the experience depends on structured input.

The design approach focused on:

grouping related fields

improving scanability

creating predictable layouts

keeping actions visible and clear

reducing overwhelm in long forms

The goal was simple: make dense input feel calm and usable.

Sender, Receiver, and Package Logic

Different types of shipment data were separated into dedicated views.

Sender and receiver information each have their own focused step, which improves readability and lowers the chance of mistakes.

Package details were treated as a separate operational layer, making it easier to handle dimensions, quantity, and shipping-specific parameters without mixing them with contact data.

Document Workflows

The product also includes several document-driven workflows:

labels and tracking

packing lists

waybills

These areas rely on structured table views that support quick review, editing, downloading, and deletion.

Consistency across these modules helps the system feel more stable and efficient in repeated use.

Settings and Technical Tools

One of the strengths of the product is that it goes beyond operational shipment tasks.

The system also supports:

workstation settings

printer configuration

API-related setup

developer-facing documentation

This added another level of complexity and required a design approach that could support both operational users and technical needs inside one product ecosystem.

Design Approach

The interface is built around clarity and operational speed.

Key design decisions include:

strong side navigation

large structured surfaces

tab-based progression

clear primary and destructive actions

consistent visual logic across screens

The UI is intentionally practical.
Its strength comes from structure, not excess.

Outcome

The result is a clearer enterprise interface for shipping and document management.

By breaking complex workflows into focused modules and step-based screens, the platform becomes easier to navigate, easier to complete, and more useful in real working conditions.

This project demonstrates the ability to design for:

enterprise workflows

complex form systems

document-heavy interfaces

clarity under operational complexity
